Directives

Attribute Directives

ngClass

Used for adding/removing many class at the same time.
The ngClass directive takes a key:value control object.
Each key is a CSS class name, and each value should be true
if the class is to be added, or false if it is to be removed.

ngStyle

Used for adding/removing many styles at the same time.
The ngStyle directive takes a key:value control object.
Each key is a style name, and each value is whatever is appropriate for that style.

ngModel

** Must import the FormsModule into your ngModule to use ngModel. **
It is actually comprised of a [ngModel] input & (ngModelChange) output property.
[(ngModel)] can only be used to set a data-bound property.
Use the separated properties when you need to perform
something different or more complex.

Structural Directives

ngIf

Used to add/remove elements from the DOM. Must be bound to a condition expression.
Important to note that this is not the same as showing/hiding the element.

example usage:
< app-component *ngIf = "someExpression" > < app-component/>

I am some text within an ngIf directive.

We can use ngIf directives to protect guard elements against null values.

example:
< div *ngIf="realObject" > {{ realObject.value }} < /div>

ngSwitch

Used just like a typical switch statement. It will display one element from
several possible elements based on a switch condition.

example usage:

< div [ngSwitch]="someObject.color" >
< app-red
*ngSwitchCase="'red'" > < /app-red >
< app-blue
*ngSwitchCase="'blue'" > < /app-blue >
< app-green
*ngSwitchCase="'green'" > < /app-green >
< app-unknown
*ngSwitchDefault > < /app-unknown >
< /div >

example:

I'm the red case!
I'm the green case!
I'm the blue case!
I'm the default case!

ngForOf

Used to present a list of items. A block of HTML is defined that defines how a single item should be displayed.
That block is then used as a template for the rest of items in the list.

example usage:
< div *ngFor="let item of items" > {{ item }} < /div >

example:
Component data: listOfItems = [ 'item1', 'item2', 'item3', 'item4', 'item5' ]

Output with ngFor:

Element {{ i }} is: {{ item }}

Pipes

Sometimes, the result of an expression or a value might need some transformation
before using it in a binding. You might need to display a number as currency,
force uppercase text, or filter/sort a list. These small transformations can be
done using pipes, simple functions that accept an input and return a transformed value.
They can be applied within template expressions using the pipe operator: (|)

example usage:
< /div > Text through uppercase pipe: {{ someText | uppercase }} < /div >

examples:

original: I'm some text!
after pipe | uppercase : {{ "I'm some text!" | uppercase }}

original: 123456
after pipe | currency : {{ 123456 | currency }}

original: {{ today }}
after pipe | date:'longDate' : {{ today | date:'longDate' }}

original: Object
after pipe: | json : {{ jsonPipe | json }}
